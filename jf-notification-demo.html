How to Implement

Adding the notification system to your application is simple. Just add the following script tag to your HTML:

<script src="jf-notification-client.js"></script>
<script>
  // Initialize the notification client
  const client = new JFNotificationClient({
    apiKey: 'your-api-key',        // Your API key for authentication (if required)
    autoConnect: true,             // Automatically connect on initialization (default: true)
    serverUrl: 'wss://example.com' // WebSocket server URL (default: auto-detect)
  });
  
  // Create a notification bell in a specific element
  client.createNotificationBell(document.getElementById('notification-placeholder'));
  
  // Listen for notifications
  client.onNotification(function(notification) {
    console.log('New notification:', notification);
  });
</script>

Sending Notifications

On the server side, you can send notifications using the Notification API:

// Example using fetch
fetch('/api/notifications', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer your-api-key'
  },
  body: JSON.stringify({
    type: 'success',
    title: 'Order Complete',
    description: 'Your custom frame order #3842 has been completed and is ready for pickup!',
    recipients: ['user-123'] // Optional - send to specific users
  })
});

API Reference

Client API

The JFNotificationClient class provides the following methods:

  • connect() - Connect to the notification server
  • disconnect() - Disconnect from the notification server
  • onNotification(callback) - Register a callback for notifications
  • offNotification(id) - Remove a notification listener
  • getNotifications() - Get all notifications
  • getUnreadCount() - Get unread notification count
  • markAllAsRead() - Mark all notifications as read
  • markAsRead(id) - Mark a specific notification as read
  • createNotificationBell(element) - Create a notification bell in the specified element

Server API

The notification server provides the following endpoints:

  • POST /api/notifications - Send a notification
  • GET /api/notifications - Get all notifications
  • GET /api/notifications/unread - Get unread notifications
  • PUT /api/notifications/:id/read - Mark a notification as read
  • PUT /api/notifications/read-all - Mark all notifications as read
